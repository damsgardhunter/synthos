# MatSci-Ōł× Supercomputer ŌĆö Materials Science AI Platform

## Overview
MatSci-Ōł× is an AI-powered supercomputer platform dedicated to accelerating the discovery of room-temperature superconductors (Tc >= 293K). It integrates advanced AI for natural language processing, novel material generation, and machine learning predictions to revolutionize material discovery. The platform aims to continuously learn, expand its knowledge base, and offers significant market potential across high-tech industries.

## System Architecture

### Stack
- **Frontend**: React, Vite, TanStack Query, wouter
- **Backend**: Express.js, TypeScript
- **Database**: PostgreSQL (via Drizzle ORM)
- **UI**: shadcn/ui, Tailwind CSS, Recharts
- **AI**: OpenAI gpt-4o-mini (via Replit AI Integrations)
- **Real-time**: WebSocket (ws)

### Startup Architecture
- `server/index.ts` opens port 5000 IMMEDIATELY with only lightweight imports (express, http)
- All heavy imports (routes, seed, storage, vite, learning modules) are loaded DYNAMICALLY inside `initializeApp()` callback after port opens
- This allows the workflow manager to see the port within seconds, while full initialization (~3-4 min) runs in background
- A `/api/health` endpoint is available immediately even during initialization
- TypeScript compilation of learning modules takes ~85s (registerRoutes), Vite setup takes ~150s

### Core Features
- **AI-Driven Learning Engine**: Orchestrates 13 distinct learning phases, including multi-fidelity screening and novel synthesis reasoning.
- **ML Prediction Engine**: Combines gradient boosting and OpenAI gpt-4o-mini for superconductor scoring.
- **ML Governance** (`model-llm-controller.ts`): Architecture selection heuristic favors XGBoost for <100 samples, 70/30 XGBoost/GNN ensemble for 200-500 samples (GNN has too many parameters to be primary at this scale), and allows GNN-primary or balanced ensemble for >1000 samples. Architecture reassessment triggers at dataset bucket thresholds [128, 256, 512, 1024, 2048, 4096]. LLM feature proposals include SHAP importance of active features with disable capability for zero-importance features. Custom feature enabled/disabled state is persisted to `system_state` PostgreSQL table (key-value jsonb store) and restored on server restart via `ensureFeatureStateLoaded()` called from `startEngine()`.
- **Physics Engine**: Performs deterministic calculations using a 96-element database, including advanced parameter calculations.
- **Topological Superconductor Detection**: Estimates Z2 invariant, Chern number, and Majorana feasibility. Includes Symmetry Indicator method (EBR decomposition + parity at TRIM), ML Topology Predictor (gradient-boosted composition model), and TSC Combined Score (Tc + topology + interface potential). TopologyŌåÆPairing feedback: when Z2 is non-trivial (>0.5) or Weyl nodes detected (diracNodeProb>0.4 or class="Weyl-semimetal"), topology-engine feeds into pairing-mechanisms to boost Triplet/Odd-Parity channel weights and override pairing symmetry (triplet, p+ip, odd-parity).
- **Quantum Pairing Mechanism Simulator**: Analyzes 7 pairing channels + topological channel. Accepts optional `TopologicalAnalysis` to avoid redundant topology computation and to apply topological boost (Triplet/Odd-Parity weighting). When boost is applied, `topologicalBoostApplied=true` and `tripletOddParityWeight` reports combined spin+topo weight.
- **KineticŌåÆSynthesis Gate Integration**: `evaluateSynthesisGate()` accepts optional `KineticInput` (kineticScore, lifetime, strategies). When kineticScore<0.3, a penalty reduces composite synthesis score; when kineticScore<0.15 with no stabilization strategies, a hard rejection reason is added. Wired in `insertCandidateWithStabilityCheck` so kinetic results flow directly into synthesis feasibility.
- **GNN-Derived Anisotropy**: `computeElectronicDimensionalityConstraint` in advanced-constraints.ts now uses `deriveGNNAnisotropy()` which extracts the dimensionality, topology, and structure segments from `encodeGenome()` (materials-genome.ts) to compute anisotropy from 3D lattice connectivity features rather than simple hardcoded ratios. Falls back to heuristic-based anisotropy only if genome encoding fails. Result field `gnnDerived: boolean` tracks which method was used.
- **K-Point Resolved Phonon Soft Modes**: `computePhononSoftModeConstraint` now calls `computePhononDispersion()` to analyze the full phonon dispersion curve segment by segment (╬ō-X, X-M, M-╬ō). Identifies which k-point segments have soft modes, applies physics-based coupling boosts (X-M: 1.35x, M-╬ō: 1.25x) reflecting stronger coupling at zone-boundary high-symmetry points. New fields: `kPointResolved`, `softModeKPoints`, `dominantSoftKPoint`, `kPointCouplingBoost`.
- **Materials Genome Representation**: Utilizes a 256-dimensional latent vector for similarity search and inverse design.
- **Autonomous Theory Discovery Engine**: Uses symbolic regression to uncover mathematical relationships.
- **Self-Improving Physics Models**: Parameterized physics equations optimized via Bayesian methods.
- **Generator Resource Manager**: Centralized allocation and adaptive rebalancing for 5 candidate generators.
- **Synthesis Pathway Modeling & Optimizer**: Simulates multi-step reaction pathways and optimizes synthesis conditions.
- **Experimental Validation Planner**: Ranks candidates, generates synthesis instructions, and suggests characterization methods.
- **Autonomous Hypothesis Engine**: AI for generating, testing, and ranking superconductivity theories.
- **Discovery Memory System**: Pattern-based learning for high Tc discoveries. Cluster-mining penalty (similarity > 0.85 ŌåÆ -0.15 penalty) and novelty bonus (similarity < 0.4 ŌåÆ +0.1 bonus).
- **Frontier Discovery RL Bonus**: `getFrontierDiscoveryBonus()` in `landscape-intelligence.ts` checks if a formula falls in a frontier region (element overlap + family match ├Ś exploration score). Materials discovered in frontier regions receive up to 25% RL reward bonus, encouraging exploration of unexplored chemical space gaps. Integrated into the RL element outcome recording in the autonomous loop.
- **Throughput Watchdog**: Monitors candidates/hr vs baseline (established at cycle 20). If throughput drops below 50% of baseline during "excited" tempo (indicating resource leak), automatically prunes `alreadyScreenedFormulas` (30% oldest entries when >80% of 50K cap), clears stale `dftEnrichmentTracker` entries (40% oldest when >5000), and clears `recentLogCache`. One-shot per degradation episode; auto-resets when throughput recovers.
- **Gradual Mutation Cooldown**: Tracks `currentMutationLevel` and `peakMutationLevel`. After stagnation-driven escalation (levels 2ŌåÆ3ŌåÆ4), Tc improvement triggers gradual step-down (peak - cyclesSinceImproved) rather than immediate reset to level 1. Prevents losing momentum in newly productive chemical regions.
- **Constraint-Relaxed Candidate Isolation**: Candidates inserted during `explorationModeActive=true` are tagged with `constraintRelaxed: true` in `mlFeatures`. These candidates are excluded from XGBoost success training (`incorporateSuccessData`), feature DB (`buildAndStoreFeatureRecord`), and BO observations to prevent unphysical data from polluting surrogate models.
- **Graph Neural Network Surrogate**: 5-model deep ensemble with heteroscedastic heads (16-target output: 8 means + 8 learned log-variances for formation energy, phonon stability, Tc, confidence, lambda, bandgap, DOS proxy, stability probability) + latent embedding. Heteroscedastic loss: L = (y-mu)^2/sigma^2 + log(sigma^2) for Tc and lambda. MC dropout with 10 passes (rate=0.1) as Bayesian posterior approximation (50 total predictions per query: 5 models x 10 MC passes). Uncertainty decomposed into: epistemic (variance across all 50 individual predictions ŌĆö captures both inter-model and MC dropout variation), aleatoric (mean of heteroscedastic predicted variances). Total sigma = sqrt(epistemic + aleatoric) ŌĆö no separate MC term to avoid double-counting. Combined uncertainty uses dynamic weight profile: high-Tc mode (Tc>77K) weights Tc=0.35, ensemble=0.25, latent=0.15, lambda=0.15, FE=0.05, BG=0.05; standard mode weights Tc=0.25, ensemble=0.20, latent=0.15, FE=0.15, lambda=0.10, BG=0.15. Weight profile included in UncertaintyBreakdown response. Dead perturbWeights function removed (superseded by analytic backprop + MC dropout). DFT training dataset uses Map<string,number> index for O(1) formula lookup. Latent embeddings auto-refresh when DFT dataset grows >5% (via setImmediate, debounced). 95% CI = mu +/- 1.96*sigma_total. Ensemble diversity: different prime seeds per model + different bootstrap ratios (0.75/0.80/0.85/0.80/0.75). W_mlp2_var initialized with smaller scale (0.05) and bias=-2.0 for reasonable initial variance. Architecture: 4 attention message-passing layers + CGCNN conv + 3-body interactions + learnable residual gates (4 sigmoid-gated scalars) + attn/mean/max pooling. Activations: leakyReLU (alpha=0.01) throughout forward pass. Tc variance: softplus(logVar) in normalized [0,1]┬▓ space internally, scaled to K┬▓ (├Ś300┬▓) only at output for display/CI. Heteroscedastic loss operates in pure normalized space: L = tcError┬▓/tcVarNorm + log(tcVarNorm) with proper chain rule through softplus for variance gradient. Training: full analytic backpropagation through entire network ŌĆö MLP head (W_mlp2 ŌåÆ h1 ŌåÆ W_mlp1 ŌåÆ pooled) + pooling layer (mean/max/attention ŌåÆ per-node dL/dEmb) + all 4 attention message-passing layers (W_update, W_message gradients via chain rule through leakyRelu, attention aggregation, and residual gates) + CGCNN convolution (W_gate, W_value, b_gate, b_value via sigmoid/softplus derivatives through gate-value product) + input projection (W_input_proj, b_input_proj). Residual gate gradients computed via dL/dGate = sum(dL/dEmb * residual) * sigmoid'(gate). Graph layer LR = 0.1├Ś MLP LR for stability. Graph embeddings restored from cache before each forward pass. Graph cache keys use MD5 hash of structure JSON (collision-safe). Lambda targets sourced from lambda-regressor predictions (confidence Ōēź 0.5) when available, falling back to Tc/50 heuristic only when no regressor data exists. Validation: held-out set from splitTrainValidation (20%, seed=42); fallback uses tail-50 of a separately-shuffled (seed=12345) SUPERCON dataset to prevent train/validation overlap. Performance: fused matVecLeakyRelu/matVecAddLeakyRelu operations; Float32Array flat-matrix WeakMap cache for contiguous memory; invalidated after training. NODE_DIM=32, HIDDEN_DIM=48, EDGE_DIM=24, OUTPUT_DIM=16.

- **Synthesis-First Gate** (`synthesis-gate.ts`): Hard gate in the candidate pipeline that rejects materials with compositeScore < 0.2 regardless of predicted Tc. Materials classified as: trivial (one-pot, score>=0.8), one-pot (score>=0.6, <=2 steps), multi-step (score>=0.35, <=5 steps), complex (score>=0.2), impractical (score<0.2, hard-rejected). Deprioritize threshold at 0.35. Integrated as hard gate in `insertCandidateWithStabilityCheck()` (after kinetic stability, before deliberation) and into GA `genomeToRoute()` (60% gate score + 40% physics feasibility blend). - **DFT Band Analysis Bridge** (`dft/dft-band-analysis.ts`): Full DFT-driven topology classification and Fermi surface reconstruction pipeline. **Crossing Dispersion Classifier** (`classifyCrossingDispersion`): analyzes E(k) curvature at each band crossing ŌĆö classifies as "linear" (Dirac-like, curvature < 0.3*velocity), "quadratic" (parabolic), or "flat"; computes velocity (dE/dk) and SOC gap (energy splitting to adjacent band) at each crossing point. **SOC Gap Detector** (`detectSOCGaps`): scans all k-points for small gaps (1-150 meV) between adjacent bands near Fermi level ŌĆö identifies spin-orbit induced splittings. **Fermi Isosurface** (`computeFermiIsosurface`): finds E(k)=EF by linear interpolation between consecutive k-points where band energy crosses zero; produces 3D k-space crossing points with interpolated velocity vectors; counts distinct Fermi sheets (unique band indices), enclosed volume fraction, velocity anisotropy. **DFT Topology Classifier** (`classifyDFTTopology`): implements structure ŌåÆ band structure ŌåÆ detect crossings ŌåÆ classify topology chain. - **Topological Invariants Engine** (`server/physics/topological-invariants.ts`): Comprehensive topological phase classification module implementing 5 analysis channels. (1) **Band Inversion Detection** (`detectBandInversions`): orbital-resolved analysis at 6 high-symmetry k-points (Gamma, X, M, Z, R, A). Output: Z2 index (nu0; nu1,nu2,nu3), strong/weak indices, parity product, Wilson loop windings, confidence. (3) **Chern Number** (`computeChernNumber`): Berry curvature integration over discretized BZ (20x20 k-grid). (5) **Surface State Detection** (`detectSurfaceStates`): iterative surface Green's function for semi-infinite slab. Output: surface state count, Dirac cones (k-position, velocity, gap, helicity, protection), surface DOS, slab thickness. **Composite Classification**: 9 topological phases (strong-TI, quantum-anomalous-Hall, type-I/II-Weyl-semimetal, weak-TI, band-inverted, trivial, + Majorana-candidate suffix). Maps space group to TRIM labels by crystal system (cubic/hex/tet/ortho/mono/tri/rhombo). Computes parity eigenvalues from orbital character, SOC, and band inversion data. Symmetry indicator Z2 from parity product. Higher-order indicators for hexagonal (Z6) and cubic (Z4) systems. Detects obstructed atomic limit (non-decomposable EBR) and fragile topology. Outputs: nontrivial probability, TI/Weyl/Dirac/flat-band/trivial class probabilities, confidence. **TSC Combined Score** (`computeTSCScore`): Integrated topological superconductor metric combining Tc prediction (30%), topology score (25%), interface potential (15%), surface state quality (15%), and Majorana feasibility (15%). Interface potential from element chemistry + Z2/SOC/Dirac/Weyl bonuses. 4 band-topology signals detected: Dirac cone, Weyl node, band inversion, flat band. TSC classes: Majorana-hosting, chiral-TSC, time-reversal-invariant TSC, Weyl-SC hybrid. Threshold: TSC candidate when score>0.35, topoScore>0.2, and Z2/Weyl/Chern nontrivial. All three integrated into main `computeTopologicalInvariants()` and stored in engine mlFeatures.topologicalInvariants. API: `/api/topological-invariants/stats`, `/api/topological-invariants/:formula`, `/api/band-inversion/:formula`, `/api/surface-states/:formula`, `/api/symmetry-indicators/:formula`, `/api/ml-topology/:formula`, `/api/tsc-score/:formula`.
