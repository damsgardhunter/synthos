# MatSci-∞ Supercomputer — Materials Science AI Platform

## Overview
MatSci-∞ is an AI-powered supercomputer platform dedicated to accelerating the discovery of room-temperature superconductors (Tc >= 293K). It integrates advanced AI for natural language processing, novel material generation, and machine learning predictions to revolutionize material discovery. The platform aims to continuously learn, expand its knowledge base, and offers significant market potential across high-tech industries.

## User Preferences
- No emoji in UI text
- Use FileText icon (not ScrollText) throughout
- NEVER use "breakthrough" or "confirmed" status for superconductor predictions – all are theoretical until lab verification.

## System Architecture

### Stack
- **Frontend**: React, Vite, TanStack Query, wouter
- **Backend**: Express.js, TypeScript
- **Database**: PostgreSQL (via Drizzle ORM)
- **UI**: shadcn/ui, Tailwind CSS, Recharts
- **AI**: OpenAI gpt-4o-mini (via Replit AI Integrations)
- **Real-time**: WebSocket (ws)

### Startup Architecture
- `server/index.ts` opens port 5000 IMMEDIATELY with only lightweight imports (express, http)
- All heavy imports (routes, seed, storage, vite, learning modules) are loaded DYNAMICALLY inside `initializeApp()` callback after port opens
- This allows the workflow manager to see the port within seconds, while full initialization (~3-4 min) runs in background
- A `/api/health` endpoint is available immediately even during initialization
- TypeScript compilation of learning modules takes ~85s (registerRoutes), Vite setup takes ~150s

### Core Features
- **AI-Driven Learning Engine**: Orchestrates 13 distinct learning phases, including multi-fidelity screening and novel synthesis reasoning.
- **ML Prediction Engine**: Combines gradient boosting and OpenAI gpt-4o-mini for superconductor scoring.
- **ML Governance** (`model-llm-controller.ts`): Architecture selection heuristic favors XGBoost for <100 samples, 70/30 XGBoost/GNN ensemble for 200-500 samples (GNN has too many parameters to be primary at this scale), and allows GNN-primary or balanced ensemble for >1000 samples. Architecture reassessment triggers at dataset bucket thresholds [128, 256, 512, 1024, 2048, 4096]. LLM feature proposals include SHAP importance of active features with disable capability for zero-importance features. Custom feature enabled/disabled state is persisted to `system_state` PostgreSQL table (key-value jsonb store) and restored on server restart via `ensureFeatureStateLoaded()` called from `startEngine()`.
- **Physics Engine**: Performs deterministic calculations using a 96-element database, including advanced parameter calculations.
- **Topological Superconductor Detection**: Estimates Z2 invariant, Chern number, and Majorana feasibility. Includes Symmetry Indicator method (EBR decomposition + parity at TRIM), ML Topology Predictor (gradient-boosted composition model), and TSC Combined Score (Tc + topology + interface potential). Topology→Pairing feedback: when Z2 is non-trivial (>0.5) or Weyl nodes detected (diracNodeProb>0.4 or class="Weyl-semimetal"), topology-engine feeds into pairing-mechanisms to boost Triplet/Odd-Parity channel weights and override pairing symmetry (triplet, p+ip, odd-parity).
- **Quantum Pairing Mechanism Simulator**: Analyzes 7 pairing channels + topological channel. Accepts optional `TopologicalAnalysis` to avoid redundant topology computation and to apply topological boost (Triplet/Odd-Parity weighting). When boost is applied, `topologicalBoostApplied=true` and `tripletOddParityWeight` reports combined spin+topo weight.
- **Kinetic→Synthesis Gate Integration**: `evaluateSynthesisGate()` accepts optional `KineticInput` (kineticScore, lifetime, strategies). When kineticScore<0.3, a penalty reduces composite synthesis score; when kineticScore<0.15 with no stabilization strategies, a hard rejection reason is added. Wired in `insertCandidateWithStabilityCheck` so kinetic results flow directly into synthesis feasibility.
- **GNN-Derived Anisotropy**: `computeElectronicDimensionalityConstraint` in advanced-constraints.ts now uses `deriveGNNAnisotropy()` which extracts the dimensionality, topology, and structure segments from `encodeGenome()` (materials-genome.ts) to compute anisotropy from 3D lattice connectivity features rather than simple hardcoded ratios. Falls back to heuristic-based anisotropy only if genome encoding fails. Result field `gnnDerived: boolean` tracks which method was used.
- **K-Point Resolved Phonon Soft Modes**: `computePhononSoftModeConstraint` now calls `computePhononDispersion()` to analyze the full phonon dispersion curve segment by segment (Γ-X, X-M, M-Γ). Identifies which k-point segments have soft modes, applies physics-based coupling boosts (X-M: 1.35x, M-Γ: 1.25x) reflecting stronger coupling at zone-boundary high-symmetry points. New fields: `kPointResolved`, `softModeKPoints`, `dominantSoftKPoint`, `kPointCouplingBoost`.
- **Materials Genome Representation**: Utilizes a 256-dimensional latent vector for similarity search and inverse design.
- **Autonomous Theory Discovery Engine**: Uses symbolic regression to uncover mathematical relationships.
- **Self-Improving Physics Models**: Parameterized physics equations optimized via Bayesian methods.
- **Generator Resource Manager**: Centralized allocation and adaptive rebalancing for 5 candidate generators.
- **Synthesis Pathway Modeling & Optimizer**: Simulates multi-step reaction pathways and optimizes synthesis conditions.
- **Experimental Validation Planner**: Ranks candidates, generates synthesis instructions, and suggests characterization methods.
- **Autonomous Hypothesis Engine**: AI for generating, testing, and ranking superconductivity theories.
- **Discovery Memory System**: Pattern-based learning for high Tc discoveries. Cluster-mining penalty (similarity > 0.85 → -0.15 penalty) and novelty bonus (similarity < 0.4 → +0.1 bonus).
- **Frontier Discovery RL Bonus**: `getFrontierDiscoveryBonus()` in `landscape-intelligence.ts` checks if a formula falls in a frontier region (element overlap + family match × exploration score). Materials discovered in frontier regions receive up to 25% RL reward bonus, encouraging exploration of unexplored chemical space gaps. Integrated into the RL element outcome recording in the autonomous loop.
- **Throughput Watchdog**: Monitors candidates/hr vs baseline (established at cycle 20). If throughput drops below 50% of baseline during "excited" tempo (indicating resource leak), automatically prunes `alreadyScreenedFormulas` (30% oldest entries when >80% of 50K cap), clears stale `dftEnrichmentTracker` entries (40% oldest when >5000), and clears `recentLogCache`. One-shot per degradation episode; auto-resets when throughput recovers.
- **Gradual Mutation Cooldown**: Tracks `currentMutationLevel` and `peakMutationLevel`. After stagnation-driven escalation (levels 2→3→4), Tc improvement triggers gradual step-down (peak - cyclesSinceImproved) rather than immediate reset to level 1. Prevents losing momentum in newly productive chemical regions.
- **Constraint-Relaxed Candidate Isolation**: Candidates inserted during `explorationModeActive=true` are tagged with `constraintRelaxed: true` in `mlFeatures`. These candidates are excluded from XGBoost success training (`incorporateSuccessData`), feature DB (`buildAndStoreFeatureRecord`), and BO observations to prevent unphysical data from polluting surrogate models.
- **Graph Neural Network Surrogate**: 5-model deep ensemble with heteroscedastic heads (16-target output: 8 means + 8 learned log-variances for formation energy, phonon stability, Tc, confidence, lambda, bandgap, DOS proxy, stability probability) + latent embedding. Heteroscedastic loss: L = (y-mu)^2/sigma^2 + log(sigma^2) for Tc and lambda. MC dropout with 10 passes (rate=0.1) as Bayesian posterior approximation (50 total predictions per query: 5 models x 10 MC passes). Uncertainty decomposed into: epistemic (variance across all 50 individual predictions — captures both inter-model and MC dropout variation), aleatoric (mean of heteroscedastic predicted variances). Total sigma = sqrt(epistemic + aleatoric) — no separate MC term to avoid double-counting. Combined uncertainty uses dynamic weight profile: high-Tc mode (Tc>77K) weights Tc=0.35, ensemble=0.25, latent=0.15, lambda=0.15, FE=0.05, BG=0.05; standard mode weights Tc=0.25, ensemble=0.20, latent=0.15, FE=0.15, lambda=0.10, BG=0.15. Weight profile included in UncertaintyBreakdown response. Dead perturbWeights function removed (superseded by analytic backprop + MC dropout). DFT training dataset uses Map<string,number> index for O(1) formula lookup. Latent embeddings auto-refresh when DFT dataset grows >5% (via setImmediate, debounced). 95% CI = mu +/- 1.96*sigma_total. Ensemble diversity: different prime seeds per model + different bootstrap ratios (0.75/0.80/0.85/0.80/0.75). W_mlp2_var initialized with smaller scale (0.05) and bias=-2.0 for reasonable initial variance. Architecture: 4 attention message-passing layers + CGCNN conv + 3-body interactions + learnable residual gates (4 sigmoid-gated scalars) + attn/mean/max pooling. Activations: leakyReLU (alpha=0.01) throughout forward pass. Tc variance: softplus(logVar) in normalized [0,1]² space internally, scaled to K² (×300²) only at output for display/CI. Heteroscedastic loss operates in pure normalized space: L = tcError²/tcVarNorm + log(tcVarNorm) with proper chain rule through softplus for variance gradient. Training: full analytic backpropagation through entire network — MLP head (W_mlp2 → h1 → W_mlp1 → pooled) + pooling layer (mean/max/attention → per-node dL/dEmb) + all 4 attention message-passing layers (W_update, W_message gradients via chain rule through leakyRelu, attention aggregation, and residual gates) + CGCNN convolution (W_gate, W_value, b_gate, b_value via sigmoid/softplus derivatives through gate-value product) + input projection (W_input_proj, b_input_proj). Residual gate gradients computed via dL/dGate = sum(dL/dEmb * residual) * sigmoid'(gate). Graph layer LR = 0.1× MLP LR for stability. Graph embeddings restored from cache before each forward pass. Graph cache keys use MD5 hash of structure JSON (collision-safe). Lambda targets sourced from lambda-regressor predictions (confidence ≥ 0.5) when available, falling back to Tc/50 heuristic only when no regressor data exists. Validation: held-out set from splitTrainValidation (20%, seed=42); fallback uses tail-50 of a separately-shuffled (seed=12345) SUPERCON dataset to prevent train/validation overlap. Performance: fused matVecLeakyRelu/matVecAddLeakyRelu operations; Float32Array flat-matrix WeakMap cache for contiguous memory; invalidated after training. NODE_DIM=32, HIDDEN_DIM=48, EDGE_DIM=24, OUTPUT_DIM=16.
- **Reinforcement Learning Chemical Space Agent**: REINFORCE policy gradient agent learning optimal elements, stoichiometries, and structures.
- **Bayesian Optimization for Chemical Space Search**: Gaussian Process surrogate with mixed acquisition functions.
- **Hydrogen Network Percolation Filter**: Geometric H-network connectivity check in `hydrogen-network-engine.ts`. `checkHydrogenPercolation(atoms, lattice, cutoff=1.5)` uses union-find with periodic boundary conditions to determine if H atoms form a connected 3D network. If H atoms in a superhydride (H count >= 4) do NOT percolate (isolated clusters instead of continuous cage), `compositeSCScore` is penalized by 50%. The geometric check overrides the formula-based proxy (`coordination × H-fraction > 2.0`) when atom positions are available. Integrated at crystal generation time (CDVAE and distribution-based diffusion paths in `engine.ts`) and in the hydrogen network analysis pipeline. Percolation results (`percolates`, `clusterCount`, `largestClusterFraction`) are stored in `hydrogenNetwork` ML features and candidate notes.
- **Crystal Diffusion Generator**: Generates novel crystal structures by refining random atomic positions.
- **Prototype-Free Lattice Generator + Evolutionary Search**: Generates crystal structures from scratch via Bravais lattice sampling + constrained atom placement (no prototype templates). Uses 7 lattice types (cubic/tetragonal/hexagonal/orthorhombic/monoclinic/trigonal/triclinic) with volume estimation from atomic volumes, minimum distance enforcement (0.7 × (r_cov_A + r_cov_B)), and optional high-symmetry site seeding. Includes evolutionary layer: population of 50, tournament selection (k=3), 4 mutation operators (lattice distortion, position perturbation, combined, atom swap), crossover (blended lattice + positions), fitness based on volume/atom, Bravais type, element diversity, and bond distance quality. Integrated as `lattice_free` strategy in generative engine (60% random + 40% evolutionary) and as fallback in QE-worker when no prototype matches. File: `server/crystal/lattice-generator.ts`.
- **Active Learning Loop**: Uncertainty-driven DFT selection and GNN surrogate retraining.
- **Real DFT Backend (GFN2-xTB)**: Integrates xtb for quantum-mechanical properties, geometry optimization, and phonon calculations. Phonon calculator uses finite displacement method with 0.01 Å step (converted to Bohr for xTB gradient consistency). Force constant matrix entries clamped at ±50 Ha/Bohr² to prevent numerical explosions. Frequencies clamped at ±5000 cm⁻¹ — any |freq| ≥ 5000 is marked as artifact and excluded from DOS/omega_log/lambda calculations. Eigenvalue→frequency conversion: sqrt(|eigenvalue|) × 521.471 cm⁻¹ (Ha/(Bohr²·amu) units). Artifact detection covers both negative (<-2000) and positive (≥5000) extremes.
- **Full DFT Backend (Quantum ESPRESSO 7.2)**: Executes pw.x and ph.x calculations for top candidates via async job queue, with multi-layer reliability checks and pre-filters. Pressure-aware: known superhydrides (LaH10, YH9, CaH6, etc.) and high-H/metal-ratio compounds are tagged with estimated pressure (GPa), lattice is compressed via Birch-Murnaghan EOS before position generation, and vc-relax uses `press = P_GPa * 10` kbar in &CELL block. Geometry validation thresholds (min volume/atom, min pair distance) are relaxed for high-pressure candidates using pressure-dependent scaling.

- **Self-Improving Design Lab**: Strategy-level optimization that evolves design architectures across 8 concurrent strategy types.
- **Symbolic Physics Discovery Layer**: 12-component subsystem for automated physics equation discovery.

- **Theory Bias Safety Reset**: `theory-guided-generator.ts` tracks pre-bias/post-bias performance (pass rate + best Tc). If pass rate drops >40% or best Tc drops >25% over 3+ evaluation windows, `resetTheoryBias()` discards the current bias, `resetToDefaultWeights()` reverts generator weights to baseline. Safety stats (reset count, discarded theories) logged. Prevents false-correlation theories from poisoning the generator pipeline.

When massive-gen candidates would push HQ ratio below 20%, massive-gen is trimmed. Prevents a single dominant generator from drowning out higher-quality sources.

CDVAE candidates scored by composite `(predictedTc/300)*0.6 + noveltyScore*0.4`. Phase 11 seeds: top-Tc, mid-rank, and 10th-best for latent space diversity. Per-seed errors caught independently.
- **Semantic Insight Deduplication**: Uses OpenAI text-embedding-3-small for semantic deduplication.
- **Cross-Engine Intelligence Hub**: Central data bus (`cross-engine-hub.ts`) collecting outputs from all 9 sub-engines (topology, fermi, pairing, pressure, defect, physics, ML, synthesis, theory) with pattern aggregation and multi-engine convergence detection.
- **Novel Synthesis Path Discovery**: Evolutionary synthesis optimizer (`synthesis-discovery.ts`) using genetic algorithm with multi-engine fitness function to discover novel multi-step synthesis routes.
- **Graph-Based Synthesis Planner**: Full synthesis planning system with 4 modules:
  - `reaction-templates.ts`: 19 typed reaction templates (DAC hydrides, arc melting, solid-state, CVD, PLD, MBE, ball milling, sol-gel, flux growth, SPS, etc.) with `matchTemplates()` scoring.
  - `precursor-database.ts`: 150+ precursors covering 65+ elements with availability/cost scoring and `findBestPrecursors()`.
  - `thermodynamic-feasibility.ts`: Reaction feasibility using DFT formation energies or Miedema fallback, Gibbs free energy, kinetic barriers, Arrhenius rates, metastable quench assessment.
  - `synthesis-planner.ts`: Graph-based route planner with multi-step decomposition, composite scoring (thermo 30% + template 30% + precursor 20% + complexity 20%), `planSynthesisRoutes()` and `planAndTrack()` with stats. Persists to `synthesis_processes` and `chemical_reactions` tables.
  - Integrated into Phase 13 (synthesis reasoning) for all eligible candidates. Synthesis gate lowered from (stage>=4 AND Tc>=100) to (stage>=2 OR Tc>=20).
  - API: `/api/synthesis-planner/stats`, `/api/synthesis-planner/routes/:formula`, `/api/synthesis-planner/recent`.
- **Heuristic Synthesis Generator**: Rule-based route generator (`heuristic-synthesis-generator.ts`) with 11 chemical rules: hydrogenation (metal + H₂), alloying (arc melting), solid-state reaction, precursor decomposition (carbonate/nitrate → oxide + CO₂), sol-gel, ball milling, boride synthesis (borothermal reduction), chalcogenide (sealed tube), pnictide (sealed tube), thin-film (PLD/sputtering), nitride (ammonolysis). Each rule generates detailed step-by-step procedures, precursor lists, conditions, and confidence scores. Integrated into Phase 13 alongside graph-based planner. API: `/api/heuristic-synthesis/routes/:formula`, `/api/heuristic-synthesis/stats`.
- **ML Synthesis Predictor**: Gradient boosting model (`ml-synthesis-predictor.ts`) predicting reaction feasibility (0-1) from 20 features: weighted electronegativity, EN std, Miedema formation energy, coordination number estimate, Shannon entropy, element count, atom count, VEC, atomic radius mismatch, melting point range/avg/max, atomic radius, bulk modulus, transition metal/rare earth/H/O flags, H fraction, metal fraction. Trained on 98 synthetic entries (known feasible + infeasible compounds). Post-prediction corrections for noble gases, extreme H content, Miedema energy. API: `/api/ml-synthesis/predict/:formula`, `/api/ml-synthesis/stats`, `/api/ml-synthesis/score/:formula`.
- **Synthesis-First Gate** (`synthesis-gate.ts`): Hard gate in the candidate pipeline that rejects materials with compositeScore < 0.2 regardless of predicted Tc. Composite score = 40% ML feasibility + 30% (1 - chemical distance) + 15% precursor availability + 15% one-pot bonus + 5% non-toxic bonus - 5% toxicity penalty. Chemical distance computed from precursor-database.ts: availability penalty (35%), step count penalty (25%), toxicity penalty (25%), cost penalty (15%). Toxicity penalties for 15 hazardous elements (As, Cd, Hg, Pb, Tl, Be, U, Th, etc.) scaled by atomic fraction. Materials classified as: trivial (one-pot, score>=0.8), one-pot (score>=0.6, <=2 steps), multi-step (score>=0.35, <=5 steps), complex (score>=0.2), impractical (score<0.2, hard-rejected). Deprioritize threshold at 0.35. Integrated as hard gate in `insertCandidateWithStabilityCheck()` (after kinetic stability, before deliberation) and into GA `genomeToRoute()` (60% gate score + 40% physics feasibility blend). Routes with feasibilityScore < 0.2 filtered from GA top results. Tracks stats: evaluations, rejections, rejection rate, classification counts, recent rejections. API: `/api/synthesis-gate/evaluate/:formula`, `/api/synthesis-gate/stats`. Dashboard: "Synthesis-First Gate" panel showing pass/reject counts, rejection rate, classifications, rejection reasons, recent rejections.
- **Reaction Network Graph** (`server/synthesis/reaction-network.ts`): Directed graph model replacing rule-based synthesis step counting with graph-theoretic pathway optimization. Nodes = chemical species (precursors, intermediates, target compounds). Edges = chemical reactions with weights incorporating Gibbs free energy (Miedema model), kinetic barriers (Tammann temperature), precursor availability, temperature, and pressure costs. Dijkstra shortest-path algorithm finds optimal multi-step synthesis routes through the reaction space. `buildReactionNetwork(formula)` constructs the graph from precursor database, generates intermediate binary compound nodes, and ranks all routes by total path cost. Graph path cost integrated into synthesis gate composite score as a bonus factor. Tracks stats: networks built, nodes/edges created, avg path cost, method/family breakdowns. Dashboard: "Reaction Network Graph" panel. API: `/api/synthesis/reaction-network/:formula`, `/api/synthesis/reaction-network/stats`.
- **Adaptive Volume DNN** (`server/crystal/volume-predictor-dnn.ts`): 3-layer MLP neural network (input→64→32→1 with ReLU activations) replacing fixed radius-based volume scaling. Input: ~74-dim vector (66-dim composition features + 1 pressure + 7-dim crystal system one-hot). Self-trains from elemental lattice constants → volume data and ~30 known compound volumes (MgB2, Nb3Sn, LaH10, YBa2Cu3O7, etc.). Backpropagation with learning rate decay. Integrated into `relaxStructureAtPressure()` in pressure-engine.ts — DNN predictions replace heuristic V0_a when model is trained, with automatic fallback to original heuristic. API: `/api/volume/predict/:formula`, `/api/volume/stats`.
- **Symmetry-Augmented GNN** (`server/crystal/symmetry-subgroups.ts`): Subgroup symmetry relations database with 14 group-subgroup relations (e.g., Pm-3m → P4/mmm → Pnma), 21 Wyckoff site entries across 8 space groups, distortion vectors, and irreducible representation labels. 6-dim symmetry feature vector: Wyckoff multiplicity, site symmetry order, subgroup index, irrep encoding, distortion magnitude, parent group order. `generateBrokenSymmetryVariants()` produces lower-symmetry phases with lattice distortions and atomic displacements for exploring broken-symmetry superconducting phases. GNN node features in `graph-neural-net.ts` augmented with symmetry embeddings. Crystal generator gains `generateBrokenSymmetryCrystals()` function.
- **Uncertainty-Quantified Physics Surrogates** (`physics-engine.ts`): Allen-Dynes/McMillan Tc formulas upgraded with full uncertainty propagation. `computeTcWithUncertainty()` implements both first-order error propagation (∂Tc/∂λ, ∂Tc/∂ω_log, ∂Tc/∂μ*) and Monte Carlo sampling (500 samples). Default uncertainties: 15% on λ, 10% on ω_log, 0.02 on μ*. Returns TcWithUncertainty = {mean, std, ci95, dominant_uncertainty_source, partials, errorPropagation, MC stats}. Integrated into unified CI pipeline in `ml-predictor.ts` — every `computeUnifiedCI()` result now includes physics UQ alongside ML UQ. Dashboard: "Physics UQ" card with interactive formula input, error contribution bars, analytic vs MC comparison. API: `/api/physics/tc-uq/:formula`.
- **Retrosynthesis Engine**: Decomposition planner (`retrosynthesis-engine.ts`) generating precursor routes for target compounds. 4 route types: direct elemental combination, binary intermediate (using common binaries DB), oxide/carbonate decomposition, precursor substitution (from precursor-database.ts). Routes scored by: deltaE (Miedema), complexity, availability, similarity. Combined score: `0.4*thermo + 0.3*availability + 0.2*similarity + 0.1*complexity`. API: `/api/retrosynthesis/routes/:formula`, `/api/retrosynthesis/stats`.
- **Pressure as First-Class Feature**: `pressureGPa` is required (non-optional) in `SuperconEntry` (`supercon-dataset.ts`). All 513 training records carry explicit pressure (0 for ambient, realistic values for hydrides: LaH10=170, YH9=201, CaH6=172, H3S=155 GPa, etc.). XGBoost trains on pressure via `extractFeatures(formula, { pressureGpa })`. GNN (`graph-neural-net.ts`) accepts `pressureGpa` as a global feature injected into the pooled representation via a learned W_pressure projection. Candidate generators assign family-aware pressure via `estimateFamilyPressure(formula)` based on hydrogen ratio (≥0.7→200, ≥0.5→150, ≥0.3→100, ≥0.15→50, else 0 GPa). GA includes `mutatePressure()` (random ±20 GPa, clamped [0,350]) and `generatePressureVariants(formula, basePressure)` producing offsets at ±50/100 GPa. All candidate insertions in engine.ts use `estimateFamilyPressure()` instead of null.
- **Pressure-Aware Surrogate Models** (`pressure-aware-surrogate.ts`): Predicts Tc(P), formationEnergy(P), bandgap(P) curves using XGBoost+GNN ensemble. Tc uses Gaussian dome model (`pressureDomeFactor`, peaked at 150 GPa) instead of linear scaling. Bandgap uses exponential suppression (`pressureBandgapFactor`, exp(-P*0.003)). GNN fetched once per formula (`fetchGNNOnce`) and base features extracted once then reused across all pressure steps (composition-based features are pressure-invariant). If GNN fails, ensemble falls back to 100% XGBoost. `predictPressureCurve(formula)` normalizes formula, does coarse 10 GPa sweep (36 points), then adaptive refinement: inserts midpoint+quartile samples in intervals where |dTc/dP| > 0.25 K/GPa. Sensitivity (`isSensitive`, `sensitivityCategory`, `maxDTcDP`) computed at curve creation and stored in `PressureCurve` — eliminates O(N²) recomputation in `getPressureCurveStats`. Enthalpy PV fallback uses physically correct factor: V₀=12 Å³/atom × (1/160.2 eV/(GPa·Å³)) = 0.075 eV/(GPa·atom). Stability threshold tightened to 0.1 eV/atom. Cache keyed by `normalizeFormula()`. Cache: 500 curves, LRU eviction. API: `/api/pressure-curves/:formula`, `/api/pressure-curves/stats`, `/api/pressure-exploration/stats`.
- **Pressure-Induced Phase Transition Detector** (`pressure-phase-detector.ts`): Scans predicted properties at 18 pressure steps (0-350 GPa) via XGBoost+GNN ensemble. Detects: large dTc/dP rises/drops, bandgap closing (metallization) and opening, stability sign changes, Tc maxima (peak detection), superconducting onset. `classifyTransition()` labels as metallization, structural, superconducting_onset, tc_maximum, bandgap_closing, bandgap_opening, or stability_sign_change. 30-min TTL cache per formula. `getPhaseTransitionStats()` returns breakdown by type, confidence, pressure range distribution. API: `/api/phase-transitions/:formula`, `/api/phase-transitions/stats`.
- **Enthalpy-Based Stability** (`enthalpy-stability.ts`): H = E + P·V using Miedema formation energy + Birch-Murnaghan volume compression. `computeEnthalpy(formula, pressureGpa)` returns total energy, PV term, enthalpy, volume, bulk modulus, stability flag. `computeEnthalpyStability()` compares compound enthalpy vs decomposition products. `computeEnthalpyPressureCurve()` sweeps 17 pressure steps (0-350 GPa). `findStabilityPressureWindow()` finds pressure range where material is enthalpy-stable. Unified `STABILITY_THRESHOLD=0.025 eV` used consistently in both `computeEnthalpy` (isStable) and `computeEnthalpyStability` (isStable); `METASTABLE_THRESHOLD=0.15 eV` for metastable classification. Formation energy unit fix: `computeMiedemaFormationEnergy` returns eV/atom — no longer double-divided by totalAtoms. `computeEnthalpyStability` uses same `point.totalEnergy` as decomposition baseline (not separate Miedema call), so `diff = pv_compound - pv_elements` is the pure pressure-driven stability measure. Decomposition enthalpy fallback uses element-specific V₀ from atomic radius + Murnaghan EOS (`V=V₀·(1+B'P/B₀)^(-1/B')`) with elemental bulk modulus instead of hardcoded 10 Å³. `enthalpyCache` keyed by `normalizeFormula()` to deduplicate equivalent formulas (e.g., H10La / LaH10). Adaptive pressure refinement: after 17-point coarse grid, `findRefinementPressures` identifies stability-boundary crossings (where isStable flips) and high-gradient regions (dH/dP > 0.005 eV/GPa), inserts midpoints + quartile points; two passes (10+5 max extra points); internal dedup at 2 GPa tolerance. `getEnthalpyStats` uses tiered stability classification: elite(>=100 GPa), broad(>=50), moderate(>=20), narrow(>0), none — with `windowDistribution` breakdown. `recentResults` sorted by window width desc then minH asc (top discoveries, not random recent), field `minH_eVPerAtom` for unit clarity. Zero-copy iteration via `forEach` instead of `Array.from`. `clearEnthalpyCache(fullClear=false)` does partial clear removing entries >24h old; `clearEnthalpyCache(true)` does full wipe. Integrated into: PressurePoint (enthalpy + enthalpyStable fields), multi-fidelity pipeline stage4 stability gate (enthalpy check for P>5 GPa), phase transition detector (enthalpy_discontinuity type when dH/dP > 0.05 eV/GPa). API: `/api/enthalpy/:formula`, `/api/enthalpy/stats`.
- **Pressure-Property Maps** (`pressure-property-map.ts`): Per-material pressure response profiles storing Tc(P), stability(P), bandgap(P) curves. `buildPressureResponseProfile(formula)` creates profile from surrogate curves. `interpolateAtPressure(formula, pressure)` does linear interpolation between grid points for any arbitrary pressure. - **DFT Band Analysis Bridge** (`dft/dft-band-analysis.ts`): Full DFT-driven topology classification and Fermi surface reconstruction pipeline. **Crossing Dispersion Classifier** (`classifyCrossingDispersion`): analyzes E(k) curvature at each band crossing — classifies as "linear" (Dirac-like, curvature < 0.3*velocity), "quadratic" (parabolic), or "flat"; computes velocity (dE/dk) and SOC gap (energy splitting to adjacent band) at each crossing point. **SOC Gap Detector** (`detectSOCGaps`): scans all k-points for small gaps (1-150 meV) between adjacent bands near Fermi level — identifies spin-orbit induced splittings. **Fermi Isosurface** (`computeFermiIsosurface`): finds E(k)=EF by linear interpolation between consecutive k-points where band energy crosses zero; produces 3D k-space crossing points with interpolated velocity vectors; counts distinct Fermi sheets (unique band indices), enclosed volume fraction, velocity anisotropy. **DFT Topology Classifier** (`classifyDFTTopology`): implements structure → band structure → detect crossings → classify topology chain. - **Topological Invariants Engine** (`server/physics/topological-invariants.ts`): Comprehensive topological phase classification module implementing 5 analysis channels. (3) **Chern Number** (`computeChernNumber`): Berry curvature integration over discretized BZ (20x20 k-grid). (5) **Surface State Detection** (`detectSurfaceStates`): iterative surface Green's function for semi-infinite slab. Output: surface state count, Dirac cones (k-position, velocity, gap, helicity, protection), surface DOS, slab thickness. **Composite Classification**: 9 topological phases (strong-TI, quantum-anomalous-Hall, type-I/II-Weyl-semimetal, weak-TI, band-inverted, trivial, + Majorana-candidate suffix). Composite score: 20% inversion + 25% Z2 + 15% Chern + 20% Weyl + 20% surface states. **Engine Integration**: Engine Phase-10 computes invariants for every topology candidate alongside existing `analyzeTopology()`, stores in mlFeatures.topologicalInvariants, logs nontrivial phases. **Stats tracking**: total computations, nontrivial rate, phase breakdown, recent 50 computations. **Symmetry Indicator Method** (`computeSymmetryIndicators`): Fast topology detection from irreducible representations at TRIM points. Maps space group to TRIM labels by crystal system (cubic/hex/tet/ortho/mono/tri/rhombo). Computes parity eigenvalues from orbital character, SOC, and band inversion data. Symmetry indicator Z2 from parity product. Higher-order indicators for hexagonal (Z6) and cubic (Z4) systems. Detects obstructed atomic limit (non-decomposable EBR) and fragile topology. Outputs: nontrivial probability, TI/Weyl/Dirac/flat-band/trivial class probabilities, confidence. **TSC Combined Score** (`computeTSCScore`): Integrated topological superconductor metric combining Tc prediction (30%), topology score (25%), interface potential (15%), surface state quality (15%), and Majorana feasibility (15%). Interface potential from element chemistry + Z2/SOC/Dirac/Weyl bonuses. 4 band-topology signals detected: Dirac cone, Weyl node, band inversion, flat band. TSC classes: Majorana-hosting, chiral-TSC, time-reversal-invariant TSC, Weyl-SC hybrid. Threshold: TSC candidate when score>0.35, topoScore>0.2, and Z2/Weyl/Chern nontrivial. All three integrated into main `computeTopologicalInvariants()` and stored in engine mlFeatures.topologicalInvariants. API: `/api/topological-invariants/stats`, `/api/topological-invariants/:formula`, `/api/band-inversion/:formula`, `/api/surface-states/:formula`, `/api/symmetry-indicators/:formula`, `/api/ml-topology/:formula`, `/api/tsc-score/:formula`. ### Physics Filtering Rules
- **Phonon stability**: xTB screening uses relaxed thresholds (maxImagModes=10, lowestFreq=-1500 cm-1) to avoid premature rejection of unrelaxed structures. Extreme artifacts (< -5000 cm-1) still rejected.
- **xTB pre-optimization**: Hessian calculations always run on at least crude-optimized structures (inline --opt crude fallback if no cached optimization exists).

- **Volume scaling**: `validateAndFixStructure` uses cbrt-based scaling with bidirectional correction (scale down when volume overshoots after distance fix). Volume ratio bounds: 0.5-2.0.

Now the correct crystallographic formulas are used: cubic `a = V^(1/3)`, tetragonal `a = (V/ca)^(1/3)`, hexagonal `a = (V/(ca × √3/2))^(1/3)`, orthorhombic `a = (V/(ba × ca))^(1/3)`.
- **Orthorhombic b/a Prototype Ratios (structure-predictor.ts)**:
  - Added `PROTOTYPE_BA_RATIOS` lookup table with experimentally-derived b/a ratios (YBCO=1.016 from 3.88/3.82 Å). The hardcoded `b = a * 1.02` is replaced by prototype-specific ratios when available. For the Vegard+volume path in orthorhombic, `a` and `b` are now solved from the cell volume constraint `V = a × b × c = a³ × (b/a) × (c/a)` using both prototype ratios.
- **Goldschmidt Tolerance Feed-Forward to Pipeline (structure-predictor.ts, multi-fidelity-pipeline.ts)**:
  - `StructurePrediction` interface now carries `goldschmidtTolerance?: GoldschmidtResult` and `structuralHints?: StructuralHint[]`. `GoldschmidtResult` = `{factor, prediction, stoichiometryValid}`, `StructuralHint` = `{source, hint, severity}`. The `buildStructuralHints()` function generates actionable hints: t>1.05 → "favor hexagonal polytypes (2H/4H/6H)"; 0.95≤t≤1.05 → "start from Pm-3m"; 0.85≤t<0.95 → "initialize with Pnma or R-3c distortion"; t<0.75 → "perovskite unlikely". Also generates dimensionality hints (quasi-2D → anisotropic k-mesh), prototype hints (infinite-layer → P4/mmm, clathrate → relax H sublattice). Hints are attached in both MP and Miedema code paths, and forwarded through `updateCandidatePhysics` in the multi-fidelity pipeline via `allData.structure.structuralHints`.
- **Dedup Moved Inside Generator Functions (structural-mutator.ts)**:
  - `seenFormulas` (now `seen: Set<string>`) passed directly into `generateDistortedLattices`, `generateLayeredStructures`, `generateVacancyStructures`, `generateStrainedVariants`. Dedup happens at generation time (before object allocation), eliminating redundant result arrays and post-hoc filtering in `runStructuralMutations`.
- **Prototype Assignment Cache (structural-mutator.ts)**:
  - `runStructuralMutations` caches `assignPrototype` results in a `Map<string, PrototypeType>` (`protoCache`). Repeated formulas skip re-parsing element data and re-running classification heuristics.
- **MutationSource Lineage Tracking (structural-mutator.ts)**:
  - New `MutationSource` interface (`parentFormula`, `parentTc`) extends all result types (`DistortedLattice`, `LayeredStructure`, `VacancyStructure`, `StrainedVariant`). Every mutation output carries its parent's identity. Engine.ts inserts mutations with notes containing `parent=<formula> parentTc=<value>` for strategy-analyzer meta-learning.

- **SC-Active Element Substitution Guard (structure-predictor.ts)**:
  - `SC_ACTIVE_ELEMENTS` set (Cu, Fe, Ni, Co, Mn, Ru, Ir, Nb, V, Ti). `generateSubstitutionVariant` will not substitute out the sole SC-active transition metal in a formula (e.g., Cu in cuprates), preserving the pairing mechanism.
- **Host-Aware Intercalation Dimensionality (structure-predictor.ts)**:
  - `generateIntercalationVariant` now calls `matchPrototype()` to inherit the host's dimensionality, space group, and crystal system instead of hardcoding "quasi-2D/P6_3/mmc/hexagonal". 3D hosts (skutterudites, clathrates) correctly produce 3D intercalated variants with "cage" topology label.
- **Wyckoff Position Validation (structure-predictor.ts)**:
  - `generateNovelPrototype` validates LLM-proposed Wyckoff labels against the space group. Multiplicity is cross-referenced with `LOW_SYMMETRY_MAX_MULT` table (P1→max 1, P-1→max 2). Labels with multiplicity exceeding the space group's maximum, or non-standard format, are appended with `[speculative]` tag in `NovelPrototype.wyckoffPositions`. A log event is emitted when speculative flags are applied.
- **Canonical Formula Deduplication (structure-predictor.ts)**:
  - `canonicalizeFormula()` normalizes formulas to alphabetically-sorted canonical form before dedup Set comparison in `generateStructuralVariants`. Prevents BaCu2O3 and Cu2BaO3 from both entering the pipeline.
- **LLM Token Efficiency (structure-predictor.ts)**:
  - Novel prototype prompt: `physics_rationale` limited to "ONE sentence"; `max_completion_tokens` reduced from 800→600. Ensures `coordination_numbers` and `wyckoff_positions` are never truncated.
- **Novel Prototype Source Tagging (engine.ts)**:
  - Novel prototype insertions now use `"novel_prototype"` generator source (was `"motif_diffusion"`). Strategy-analyzer can now track and reward structural invention as a distinct generator type.
- **Novel Prototype → Combinatorial Pipeline (engine.ts)**:
  - `suggestedElements` from LLM-generated novel prototypes are now fed directly into `findOptimalRegion()` to generate a fleet of combinatorial candidates in the new prototype's chemical space. Up to 3 optimal compositions per element set are inserted with `"novel_prototype"` source tag and `[novel-prototype-combinatorial]` notes.
- **Symmetry-Enforced Lattice Ratios (structure-predictor.ts)**:
  - After clamping LLM-proposed lattice ratios, a symmetry enforcement block forces `b=a, c=a` for cubic/rhombohedral and `b=a` for tetragonal/hexagonal/trigonal. Prevents the LLM from proposing e.g. Pm-3m with a=1.0, b=1.5, c=2.0 which would crash DFT relaxation or produce unphysical GNN embeddings.
- **Stoichiometry Template Diversity (structure-predictor.ts)**:
  - `runNovelPrototypeGeneration` now uses 7 stoichiometry templates (ABO3, A2BO4, AB2X3, AB3X, ABX2, A2B2X7, A3BX) instead of hardcoding count=1 for the first element. Enables generation of La2CuO4-type prototypes where the A-site has multiplicity >1.
- **Canonical Formula Normalization in Discovery (structure-predictor.ts)**:
  - Both `runNovelPrototypeGeneration` and `runGenerativeStructureDiscovery` now canonicalize formulas via `canonicalizeFormula()` before storage lookup and insertion. Prevents BaCu2O3 vs Cu2BaO3 dedup misses.
- **Structural Enrichment with Fractional Coordinates (structure-predictor.ts)**:
  - `StructuralEnrichment` interface stores `wyckoffSites` (label + approximate fractional coords from `WYCKOFF_APPROX_COORDS` table + speculative flag), `coordinationNumbers`, and `needsExternalRelaxation` flag. Attached to `NovelPrototype.structuralEnrichment`. Provides the GNN with 3D coordinate seeds instead of bare Wyckoff labels. Unknown sites get [0,0,0] with speculative=true and needsExternalRelaxation=true.
- **Wyckoff-Driven Element Count (structure-predictor.ts)**:
  - `runNovelPrototypeGeneration` now matches the number of elements used to the number of unique Wyckoff roles defined by the LLM (instead of hardcoded slice(0,4)). Quaternary/HEA prototypes with 5 roles will use all 5 suggested elements.
- **Parallel LLM Calls (structure-predictor.ts)**:
  - `runNovelPrototypeGeneration` uses `Promise.all` to call `generateNovelPrototype` for both principles concurrently, cutting the structural invention phase latency roughly in half.
- **Exploration-Mode Candidate Selection (structure-predictor.ts)**:
  - `runGenerativeStructureDiscovery` now has a 10% probability of selecting 3 random candidates instead of the top-3 by ensembleScore. Promotes exploration of underexplored chemical space alongside the default exploitation strategy.
- **Canonical Evolutionary Formula Reconstruction (structure-predictor.ts)**:
  - `reconstructFormula` now deduplicates elements via `Set` and sorts alphabetically before joining. Prevents malformed formulas like `LaH10H1` from duplicate intercalation, and ensures consistent ordering (Ba2Cu3O7Y→Ba2Cu3O7Y not randomized) so downstream dedup and cache lookups work correctly.
- **Intercalation Duplicate Guard (structure-predictor.ts)**:
  - Evolutionary mutation m===3 (scale + intercalate) now checks `Set(elements)` before pushing the intercalant. If the scaled element was already an intercalant (e.g., H in a hydride), it increments the existing count instead of creating a duplicate array entry.
- **McMillan Tc Fitness in Evolution (structure-predictor.ts)**:
  - Each evolutionary mutant now gets a fresh Tc prediction via `extractFeatures` + McMillan equation (lambda, logOmega, muStar=0.13) instead of carrying the dead ancestor's Tc. Score weights rebalanced to 0.4 motif + 0.25 dimensionality + 0.35 Tc/400 (was 0.5/0.3/0.2 with stale Tc). Provides a valid fitness gradient for Tc improvement across generations.
