Duplicate Explosion (Biggest Issue Right Now)

Your logs show a large fraction of candidates rejected as duplicates:

REJECTED: CaZnB2C reason=duplicate
REJECTED: BaZnB2C reason=duplicate
REJECTED: SrZnB2C reason=duplicate
REJECTED: MgZnB2C reason=duplicate

What’s happening:

Your generator is producing near-identical substitutions in the same template.

Example pattern:

MZnB2C
M = Ca, Ba, Sr, Mg

These are chemically trivial substitutions, and the system is wasting cycles generating them.

Why this is bad

It reduces search-space diversity.

Right now:

43 screened
0 passed
~25 duplicates

So the generator is effectively stuck in a small compositional neighborhood.

Fix: Element Substitution Distance

Add a periodic-table distance rule.

Example:

if substitutionDistance(elementA, elementB) < threshold:
    reject generation

Distance can be computed from:

group difference
period difference
electronegativity difference
atomic radius difference

Example rule:

distance = |groupA-groupB|
         + |periodA-periodB|
         + |EN_A - EN_B|

if distance < 1.2:
    treat as duplicate

This forces the generator to explore chemically different compositions.
