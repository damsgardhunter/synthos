Critical Error: Quantum Espresso Pseudopotential Failure

Your main error:

pw.x exited with code 2:
Fortran runtime error:
Sequential READ or WRITE not allowed after EOF marker
__read_upf_new_module_MOD_read_upf_new
__read_pseudo_mod_MOD_readpp

This occurs before the SCF even begins.

It means Quantum Espresso cannot read a pseudopotential file (.UPF).

This is very common and usually caused by one of these:

Likely Causes
❌ Corrupted pseudopotential file

The UPF file ended prematurely.

Example:

La.pbe-spn-kjpaw_psl.1.0.0.UPF

may have been truncated.

❌ Wrong QE version vs pseudopotential format

Old QE versions cannot parse UPF v2 XML pseudopotentials.

Example mismatch:

QE version	UPF version
QE 6.x	UPF v1
QE 7.x	UPF v2 XML
❌ Incorrect file path

If QE opens an empty file:

pseudo_dir = './pseudo'

but the file isn't there → EOF.

Fix Checklist

Add this validation step before launching pw.x:

validatePseudo(file):

  if file_size < 5KB:
       reject

  if not contains("</UPF>"):
       reject

  if not contains("<PP_HEADER>"):
       reject
