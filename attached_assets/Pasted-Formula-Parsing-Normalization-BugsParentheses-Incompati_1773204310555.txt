Formula Parsing & Normalization BugsParentheses Incompatibility: The regex /([A-Z][a-z]?)(\d*\.?\d*)/g used in normalizeFormula and parseFormulaCounts is "flat." It will fail on complex discovery candidates that use parentheses, such as $(La,Y)H_{10}$ or $Mg(BH_4)_2$. It will treat the characters (, ), and , as non-matches and skip them, resulting in an incomplete atom count.Fix: Implement a recursive stack-based parser to expand parentheses before running the element regex.Subscript Conversion Logic: In normalizeFormula, the line:normalized = formula.replace(/[₀-₉]/g, c => String("₀₁₂₃₄₅₆₇₈₉".indexOf(c)));is a clever way to handle Unicode, but it assumes the input contains only single-digit subscripts. If a formula has a multi-digit subscript like $H_{10}$ using Unicode ($H₁₀$), the regex replaces them one by one. This works, but ensure your parseFloat logic correctly handles the concatenated string.Sorting Inconsistency: normalizeFormula sorts by electronegativity, while many external databases (Materials Project/AFLOW) use alphabetical order. If you use this normalized string as a database key, ensure all other modules (like your AFLOW Client) follow the same convention, or you will suffer from cache misses.2. Physical Property Logic & ConstraintsThe "Metal Fraction" Threshold: In computeAnalyticalFallbacks, you define isMetallic = metalFrac > 0.3.Scientific Risk: This is a very permissive threshold. Many materials with a 31% metal fraction are actually wide-bandgap insulators (e.g., certain oxides or complex salts).Improvement: You should prioritize the bandGap calculation or check for the presence of specific electronegativity spreads ($>2.5$) to override isMetallic to false even if the metal fraction is high.Formation Energy Over-simplification: formationEnergy = -0.5 * enSpread.Constraint: This is a very rough heuristic. It works for simple binary ionic compounds but fails for intermetallics or alloys where the electronegativity spread is near zero but the formation energy is strongly negative due to metallic bonding.Fix: Use a Miedema model fallback for materials where metalFrac > 0.8.Density "Hard" Packing Fraction: In estimateDensity, you use a fixed packingFraction = 0.68 (BCC).Improvement: Superconductors often crystallize in FCC ($0.74$) or complex clathrate structures ($<0.5$). Using a fixed $0.68$ can introduce a $\sim 10\%$ error in density, which propagates into the debyeTemp calculation ($T_D \propto \sqrt{B/\rho}$).
