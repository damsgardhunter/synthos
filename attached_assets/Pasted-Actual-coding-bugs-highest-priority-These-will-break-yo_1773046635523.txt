Actual coding bugs (highest priority)

These will break your pipeline regardless of the physics.

Undefined variable
[Active Learning] Error at cycle 107: ENSEMBLE_SIZE is not defined

This is a hard runtime error.

Likely causes:

const ensemble = new Array(ENSEMBLE_SIZE)

but the constant was never declared.

Fix:

const ENSEMBLE_SIZE = 5

or load from config:

const ENSEMBLE_SIZE = config.activeLearning.ensembleSize
Undefined array access
[NextGenPipeline] pillar eval error: Cannot read properties of undefined (reading 'some')

This means something like:

pillars.some(p => p.score > 0)

but pillars is undefined.

Guard it:

if (!Array.isArray(pillars)) return false

or:

pillars?.some(...)

This error repeating 15+ times suggests the pillar module never received data.

Worker authentication timeout
[DFT-Queue] Worker loop error: Authentication timed out

Likely:

API token expired

SSH key expired

cluster auth handshake failing

This can cause silent DFT job loss.

Example:

70 imaginary modes
lowest freq = -340 cm⁻¹

or worse:

198 modes below -2000 cm⁻¹
lowest = -3985 cm⁻¹

These are not real phonons.

They are finite-difference explosions.

Causes:

xTB geometry not fully optimized

displacement step too large

too few SCF iterations

highly unstable lattice

Your artifact detector is working
ARTIFACT DETECTED — 198 modes below -2000 cm⁻¹ discarded

That is good pipeline design.

But the follow-up step is inconsistent:

stable=false
freq range [-3985, -3985]

This should instead return something like:

phonon_status = unreliable

not treat the numbers as real.

3️⃣ Physics problems in generated structures

These are common in generative materials pipelines.

A. Severe lattice overlap
Ca3FeSe4 minDist=0.653 Å

That means atoms are basically overlapping.

Typical minimum:

> 1.2 Å for H
> 1.8 Å for metals

Your generator is producing collapsed lattices.

B. Lattice expansion exploding

Example:

vol/atom=25.9 → expand 2.50
vol/atom=208 → expand 1.23
generated volume = 3200 Å³
expected = 155 Å³
ratio = 14

This is a runaway scaling loop.

Your volume correction logic is unstable.

Instead of multiplicative scaling, clamp:

scale = clamp(target/current, 0.8, 1.3)
C. Kagome template mismatch
Volume ratio check FAILED for Kagome
ratio = 14

Your Kagome generator likely assumes specific atom counts, but you inserted:

Ca3FeSe4

which probably doesn't fit the template.

5️⃣ Incorrect phonon interpretation

Right now your pipeline accepted a physically unstable structure.
