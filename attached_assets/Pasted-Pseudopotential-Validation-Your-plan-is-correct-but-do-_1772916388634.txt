Pseudopotential Validation

Your plan is correct but do NOT regenerate PPs with ld1.x automatically.

Reason:

ld1.x requires:

atomic configuration
exchange functional
cutoff radii
reference states

Automatically generating these will produce bad pseudopotentials.

Better approach

Use a pre-built pseudopotential library.

Recommended:

PseudoDojo
SSSP Efficiency
PSLibrary

Example directory:

pseudo/
  H.pbe-kjpaw.UPF
  La.pbe-spn-kjpaw.UPF
  Ta.pbe-spn-kjpaw.UPF

Then validate.

Example validation:

function validatePseudopotential(file) {

  const text = fs.readFileSync(file, 'utf8')

  if (text.length < 5000) return false
  if (!text.includes("<UPF")) return false
  if (!text.includes("</UPF>")) return false
  if (!text.includes("PP_HEADER")) return false

  return true
}

If missing:

reject job

not regenerate.
